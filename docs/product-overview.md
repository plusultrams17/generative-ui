# 生成UI プロダクト概要・活用ガイド・収益化戦略

---

## 第1部：プロダクト概要 — 「生成UIとは何か」

### コンセプト

**生成UI**は、自然言語（日本語）でAIに指示するだけで、プロ品質のUIコンポーネントを即座に生成・編集・エクスポートできるWebプラットフォームです。

コーディング不要。デザイン知識不要。「ログインフォームを作って」と入力するだけで、バリデーション付きのフォームが数秒で完成します。

### 技術スタック

| レイヤー | 技術 |
|---------|------|
| フレームワーク | Next.js 16 / React 19 / TypeScript |
| AI基盤 | Vercel AI SDK v6（GPT-4o / Claude Sonnet / Gemini Flash） |
| UIライブラリ | shadcn/ui + Radix UI + Tailwind CSS v4 |
| 状態管理 | Zustand v5（27ストア、LocalStorage永続化） |
| チャート | Recharts |
| エクスポート | JSZip（プロジェクト一括DL） |

### 対応AIモデル

| モデル | プロバイダ | 特徴 |
|-------|-----------|------|
| GPT-4o | OpenAI | バランス型（デフォルト） |
| GPT-4o mini | OpenAI | 軽量・低コスト |
| Claude Sonnet 4.6 | Anthropic | 高品質コード生成 |
| Gemini 2.0 Flash | Google | 超高速レスポンス |

## 第2部：ユーザー向け活用ガイド — 「誰が、どう使えるか」

### ターゲットユーザー別ユースケース

---

### 1. ノンエンジニア（営業・企画・マーケティング）

**課題**: UIのイメージはあるが、コードが書けない。デザイナーやエンジニアへの依頼に時間がかかる。

**活用方法**:

| やりたいこと | 操作 |
|-------------|------|
| 提案書に載せるUI案を作りたい | チャットで「問い合わせフォームを作って」→ スクリーンショット or HTMLエクスポート |
| クライアントにデモを見せたい | プレゼンテーション機能でスライド化 → フルスクリーンで提示 |
| 料金プランの比較表が欲しい | 「料金プランを3つ並べて比較表にして」→ そのままコピー |
| LP案を素早く可視化したい | テンプレートから「ランディングページヘッダー」を選択 → チャットで調整 |

**具体的な指示例**:
```
ユーザー登録フォームを作って（名前、メール、パスワード、確認パスワード）
```
```
2024年の月別売上データを棒グラフで表示して
```
```
モダンなプロフィールカードを作って（アバター、名前、肩書き、SNSリンク付き）
```

---

### 2. フロントエンドエンジニア

**課題**: プロトタイプ作成に時間がかかる。繰り返しの定型UIを書くのが非効率。

**活用方法**:

| やりたいこと | 操作 |
|-------------|------|
| プロトタイプを高速作成 | チャットで概要指示 → 生成コードをコピー → 自プロジェクトに貼付 |
| コンポーネントバリエーションを量産 | 「ダークモード対応にして」「アニメーション追加して」と反復指示 |
| 画像からUI再現 | 画像アップロード → AIが構造解析 → React+Tailwindコード出力 |
| テスト・Storybook生成 | 生成コードを基にドキュメント自動生成（内蔵ユーティリティ） |

**強力な機能**:
- **コンポーザー**: 複数コンポーネントを組み合わせてページ全体を構成 → HTMLまたはNext.jsプロジェクトとしてZIPダウンロード
- **Vercelデプロイ**: 生成したUIをワンクリックで公開URL化
- **GitHub連携**: リポジトリ作成 → コード自動コミット

---

### 3. Webデザイナー / UIデザイナー

**課題**: デザインカンプからコードへの変換コストが高い。エンジニアとのコミュニケーションギャップ。

**活用方法**:

| やりたいこと | 操作 |
|-------------|------|
| デザインカンプをインタラクティブに | 画像アップロード → React実装を自動生成 |
| カラーバリエーション確認 | 「青系からオレンジ系に配色変更して」→ 即座にプレビュー |
| レスポンシブ確認 | コンテキスト適応で自動的にモバイルファースト生成 |
| コンポーネントライブラリ構築 | テンプレート保存 → マーケットプレイスで管理 |

---

### 4. フリーランス / 制作会社

**課題**: 案件管理が煩雑。提案書作成に時間がかかる。クライアントへの成果物提示が手間。

**活用方法**:

| 機能 | ビジネス活用 |
|------|-------------|
| **プロジェクト管理** | カンバンボードで案件ステータスを一元管理。予算・期限追跡 |
| **クライアント管理** | CRMとしてクライアント情報・連絡先・ステータスを管理 |
| **提案書** | 見積書・提案書を作成。明細行・税計算・プレビュー付き |
| **プレゼンテーション** | 生成UIをスライドに組み込み、フルスクリーンで提示 |
| **ブランド設定** | プロジェクトごとにカラー・フォント・ロゴを設定 |

---

### 5. 教育・研修

**課題**: UI/UXの概念を説明するのに実物がないと伝わらない。

**活用方法**:
- 講義中にリアルタイムでUI生成してデモ
- 受講者が自分の言葉でUIを作る体験学習
- 「良いフォーム」と「悪いフォーム」を生成して比較
- アクセシビリティの重要性を実際のコード差分で説明

## 第3部：全16機能の説明

### メイン機能

| 機能 | 説明 |
|------|------|
| **チャット** | AIとの対話でUI生成。4種のツール（フォーム/テーブル/チャート/カスタム）対応。画像アップロードからのUI再現も可能 |
| **ギャラリー** | 生成履歴の一覧。お気に入り登録・タグ付け・検索・再利用・共有 |
| **ショーケース** | 共有/公開されたUIを一覧表示。マーケットプレイスと共有ストアのデータを統合表示 |
| **マーケットプレイス** | UIコンポーネントの共有プラットフォーム。公開・いいね・ダウンロード。9種のシードテンプレート付き |

### ツール

| 機能 | 説明 |
|------|------|
| **テンプレート** | 13種以上の公式テンプレート（ログインフォーム、価格表、ナビバー等）。ワンクリックでチャットに送信して即生成 |
| **エージェントビルダー** | AIエージェントのワークフローをSVGキャンバス上でビジュアル構築。ドラッグ&ドロップでノード配置・接続 |
| **オーケストレーション** | 複数エージェントの連携を定義・シミュレーション実行。コーディネーター/ワーカー/レビューアーの協調動作 |
| **コンポーザー** | 複数UIコンポーネントを組み合わせてページ構成。HTML/ZIPエクスポート。Vercelデプロイ・GitHub連携 |

### 管理

| 機能 | 説明 |
|------|------|
| **プロジェクト** | カンバンボード/リスト表示。ステータス追跡・予算管理・ブランド設定 |
| **クライアント** | CRM。連絡先・ステータス・プロジェクト紐付け |
| **提案書** | 見積書作成。明細行追加・税/割引計算・PDFプレビュー |
| **プレゼンテーション** | スライドベースの発表資料。フルスクリーン表示・キーボードナビゲーション |

### システム

| 機能 | 説明 |
|------|------|
| **統計** | KPIダッシュボード・ツール分布・トレンド・キーワード分析 |
| **設定** | APIトークン管理・デフォルトモデル選択・エージェント/MCPサーバー設定・データ管理 |
| **管理者** | ユーザー管理・監査ログ・コスト管理・SSO設定 |
| **ヘルプ** | 使い方ガイド・FAQ・キーボードショートカット一覧 |

**モデル**: 無料で基本利用 → 有料で制限解除

| プラン | 月額 | 内容 |
|--------|------|------|
| **Free** | ¥0 | 月20生成 / GPT-4o miniのみ / エクスポート制限 |
| **Pro** | ¥1,980 | 月200生成 / 全モデル対応 / Vercelデプロイ / GitHub連携 |
| **Team** | ¥4,980/人 | 無制限生成 / チーム共有 / ブランド設定 / 提案書・プレゼン |
| **Enterprise** | 要相談 | SSO / 監査ログ / SLA / カスタムモデル / オンプレ対応 |

## 第5部：競合と差別化ポイント

### 競合分析

| サービス | 特徴 | 生成UIとの差 |
|---------|------|-------------|
| v0.dev (Vercel) | UIコード生成 | 英語中心。日本語最適化されていない |
| Bolt.new | フルスタック生成 | 重量級。UI単体の軽量生成には不向き |
| ChatGPT/Claude直接利用 | コード生成 | プレビューなし。ツール統合なし。ワークフロー管理なし |
| Figma AI | デザイン生成 | コード出力なし。開発者向けでない |

### 生成UIの独自優位性

1. **日本語ネイティブ** — 日本語プロンプトに最適化されたシステムプロンプト。日付形式、敬語UI、全角文字対応
2. **エンドツーエンド** — 生成→プレビュー→編集→エクスポート→デプロイが一気通貫
3. **案件管理統合** — プロジェクト/クライアント/提案書/プレゼンが一体化。制作会社の業務フローそのもの
4. **マルチモデル** — GPT-4o/Claude/Geminiを切り替え可能。コスト・品質の最適化が自由
5. **オフライン可能** — LocalStorage永続化。ネットワーク断でもギャラリー/テンプレート/設定は維持

## 第6部：解決する課題と市場機会

### 解決する課題

| 課題 | 現状 | 生成UIでの解決 |
|------|------|---------------|
| UIプロトタイプに時間がかかる | デザイン→コード変換に数時間〜数日 | 数秒で生成。反復修正も会話で即座に |
| エンジニアとデザイナーの認識ギャップ | 静的カンプと実装の乖離 | 動くUIをリアルタイムで共有 |
| ノンエンジニアがUIを作れない | Figma学習コストが高い | 日本語で話すだけ |
| テンプレート探しに時間がかかる | Web検索→カスタマイズの手間 | マーケットプレイスからワンクリック |
| 制作案件の管理が分散 | Slack/スプレッドシート/メール | プロジェクト/クライアント/提案書を統合管理 |
| AI活用の技術ハードル | API連携やプロンプト設計が必要 | GUI操作のみ。設定画面でAPIキー入力するだけ |

### 市場規模の参考値

| 市場 | 規模 |
|------|------|
| 日本国内ローコード/ノーコード市場 | 約3,000億円（2025年予測） |
| グローバルAIコード生成市場 | $5B+（2025年推定） |
| 日本のWeb制作会社数 | 約15,000社 |
| フリーランスエンジニア（日本） | 約25万人 |
